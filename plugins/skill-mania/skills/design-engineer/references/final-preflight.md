# Final Preflight

Use this before closing substantial UI implementation, redesign, public-page work, or screenshot-driven review.

## Context
- Check `company.md`, `PRODUCT.md`, `DESIGN.md`, brand notes, screenshots, and existing components when present.
- State the interface register: operational, editorial, product-inspection, portfolio, commerce, game, or another concrete fit.
- Confirm the first viewport shows the actual product, object, state, person, place, or workflow when that is what the page is about.

## Rendered Checks
- Desktop viewport: primary task visible, hierarchy clear, no unintended overlap or clipping.
- Narrow mobile viewport: navigation, actions, media, and long text still fit.
- Long content: labels, names, numbers, dates, and untranslated strings wrap or truncate intentionally.
- States: loading, empty, error, disabled, selected, hover, focus, and validation are present when the workflow needs them.
- Media: focal points are not hidden by overlays, cards, nav, or text.
- Controls: touch targets, focus order, accessible names, contrast, and reduced-motion behavior are acceptable.

## Visual Tells
- Scan for unchosen purple gradients, generic hero copy, decorative cards, oversized radius, gradient text, custom cursors, full-screen hero traps, and scroll-driven effects.
- Keep intentional exceptions only when a product, brand, or workflow reason is named.
- Replace placeholder persona names, fake screenshots, and abstract feature cards with real entities, states, examples, or objects.

## Close-Out
- Report the direction used and the evidence checked.
- Name unverified rendered states instead of implying final QA happened.
- If a visual issue remains, give the smallest concrete fix before broader polish.
