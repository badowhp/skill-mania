# Verification Checklist

Verify a revision by comparing it against the original and the brief, not by rereading it in isolation.

## Deterministic Floor
1. Run `scripts/scan-ai-slop-text.py` on substantial new or revised prose and address material findings.
2. Diff-check preserved content: every fact, number, name, quote, command, code block, and link that had to remain intact is unchanged; list each deliberate change separately.
3. Check the requested edit level was honored: a line edit did not silently restructure; a developmental pass did not skip the requested scope.
4. Verify structural promises: headings match the agreed outline, internal references point at sections that exist, and required platform elements (front matter, metadata, word limits) are met exactly.

## Evidence Checklist
- Read the full revised piece once as a reader: opening promise, through-line, and ending are checked as a whole, not per paragraph.
- Voice constraints are demonstrated with a before/after example, not asserted.
- Claims added during revision carry sources or are flagged for the author to confirm; nothing presented as fact was invented to smooth a transition.
- Unresolved factual, legal, rights, accessibility, or platform-sensitive issues are reported explicitly instead of implied as handled.
