# PMA Project Handbook

Use this reference to create or review a proportionate Austrian/German `Projekthandbuch`. Treat the PMA file as a reusable artifact catalogue, not as a complete modern delivery method or proof of pma/IPMA compliance.

## Source Status

- The user-supplied [legacy Standardprojekthandbuch.doc](https://www.pma.at/Standardprojekthandbuch.doc) is a 28-page Microsoft Word template. Its document metadata dates the file to 2018, while its footer identifies template version 2.7 dated 10 May 2011.
- PMA's [official downloads page](https://www.pma.at/de/service/downloads) lists a German `Standard Projekthandbuch` DOCX modified on 11 September 2024.
- The current direct file is [standardphbv27deutsch.docx](https://www.pma.at/files/downloads/437/standardphbv27deutsch.docx). Its internal metadata also records 11 September 2024, but its artifact structure remains substantially the version 2.7 template.
- PMA describes ICB4 and `pm baseline 3.1` as the current basis for pma/IPMA certification. The handbook template alone is not a certification standard.

Source status was checked on 11 July 2026. Recheck the official downloads page before calling any version current.

## Tailoring Rule

Start with the control need, then select the smallest useful artifacts. Do not create blank sections merely because the source template contains them.

- For a small, low-risk project, combine the charter, objectives, milestone plan, roles, RAID, status, and closure record.
- For a medium or cross-team project, add work packages or backlog, resource and cost controls, stakeholder analysis, communications, decisions, changes, and transition.
- For a high-risk, regulated, contractual, or capital project, preserve controlled baselines, traceability, formal approvals, assurance gates, acceptance evidence, and auditable change history.

## Artifact Map

| PMA artifact | Management purpose | Minimum useful content |
| --- | --- | --- |
| `Projektauftrag` | Authorise and bound the project | trigger, start and end conditions, dates, objectives, non-objectives, major phases, resource/cost envelope, sponsor, project manager, team, approval |
| `Projektzieleplan` | Make outcomes testable | primary and secondary objectives, non-objectives, measures, target values, owner, approved adaptations |
| Vor-/Nachprojektphase | Preserve context and adoption work | prior decisions and documents, lessons from similar work, post-project operations, benefits, follow-on actions |
| `Projektumwelt-Analyse` | Manage stakeholder and environmental relationships | stakeholder/environment, potential or conflict, response, owner, date, related work item |
| Project and strategy relationships | Expose external dependencies and value alignment | related initiatives, dependency or conflict, action, owner/date, organisational objective |
| `Projektorganigramm` | Define governance | sponsor, project manager, team, contributors, required skills, authority, escalation path |
| `Betrachtungsobjekteplan` | Decompose the solution or subject of work | product, service, system, organisation, or other objects and their relationships |
| `Projektstrukturplan` | Decompose total project work | complete work hierarchy, stable codes, deliverable or phase orientation, ownership |
| `Arbeitspaket-Spezifikation` | Create an executable work contract | scope, exclusions, outputs, acceptance, progress measure, owner, dependencies, estimate |
| `Projektfunktionendiagramm` | Clarify responsibility by work package | accountable execution, contributors, informed parties, and any approval or consultation rights |
| Milestone and bar plans | Control sequence and forecast | baseline, current forecast, actual dates, predecessors, decision gates, critical constraints |
| Personnel and cost plans | Control capacity and money | baseline, approved current plan, actual, forecast, variance, source, owner |
| Communication structure and team rules | Make coordination deliberate | forum, purpose, participants, cadence, owner, decision rights, records, working agreements |
| Risk analysis | Make uncertainty actionable | cause, uncertain event, effect, probability, impact, response, contingency, owner, review date |

## Lifecycle Controls

### Start

Use a start workshop to align on outcome, scope, roles, delivery approach, assumptions, risks, working agreements, and the first decision or milestone. Record unresolved points rather than converting them into implied agreement.

### Coordinate

Accept work packages or deliverables against explicit criteria. Record the item, accountable owner, acceptance evidence, accepter, date, conditions, and residual work.

### Control

Create a dated progress report that covers:

- overall status and confidence
- objectives and expected value
- deliverable or work progress
- milestones and forecast dates
- resources and costs
- stakeholders, dependencies, and external context
- team, organisation, and delivery health
- measures, decisions, changes, and escalations

Define status thresholds before assigning red, amber, or green. Do not average a critical red condition into an apparently healthy overall status.

### Close

Record acceptance, outcome and objective achievement, schedule and delivery performance, resources and cost, stakeholder and team observations, lessons, residual actions, operational handover, benefits follow-up, and formal closure approval.

## Modernise the Template for 2026

Add controls the old template does not make sufficiently explicit:

- measurable value, benefits owner, and post-project review date
- assumptions, issues, dependencies, decisions, and changes as separate controlled records
- deliverable quality and acceptance criteria
- procurement, supplier, contract, and claims obligations where relevant
- privacy, security, accessibility, sustainability, and regulatory requirements
- operational readiness, training, data migration, support, and adoption
- agile or hybrid backlog, release, iteration, and governance relationships
- AI-use register, approved tools and data, human review, provenance, and override or escalation rules when AI affects project work
